# Watchlist Entries テーブル設計 — 歴史記録 (圧縮済み)

> 2026-03 の DB 再設計メモを圧縮した歴史記録。現行仕様ではない。
> 関連: 参照先が「生記事 → 分析済み記事」へ移った経緯は
> [記事エンティティの変遷](article-entity.md) に統合済み。本書はウォッチリスト表
> 自体の設計(複合主キー化など)の記録として残す。

## 当時の決定
ウォッチリストを「1行 = 1つの保存操作」を表す不変のエンティティとして設計する。不要な
サロゲートキーを廃し、利用者と記事の組み合わせを複合主キーにして一意化する(このテーブルを
参照するものは無いので独立 ID は要らない)。利用者参照は認証側のテーブルへ、スキーマを跨いで
外部キーを張る。

## その後の帰結
この方針で実装された。なお参照先の記事は、その後「生記事」から「分析済み記事」へ張り替え
られている(記事識別子の再設計による)。
