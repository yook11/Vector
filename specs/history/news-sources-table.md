# News Sources テーブル設計 — 歴史記録 (圧縮済み)

> 関連: 再設計全体の方針(secure-by-design / 一時状態を DB 外へ)は

> ソース表固有の設計(設定の DB 外出し・統合)の記録として残す。

## 当時の決定
ソースを「管理者が収集対象として認めた少数の固定マスタ」と位置づけ、信頼性は管理者の選定で
担保する。HTTP キャッシュの一時状態、スケジューラの次回取得時刻、連続エラー数といった
一時的・運用的な状態は DB から外へ追い出し、ドメイン属性と最小限の設定だけに絞る。取得日時の
ような事実は上書きせず取得履歴側に委ねる。当初は「ドメイン用」と「パイプライン用」の2テーブル
分割も検討したが、設定の大半を DB 外へ出した結果残りがわずかになり、統合した。

## その後の帰結
この方針で再設計され、定着した。
