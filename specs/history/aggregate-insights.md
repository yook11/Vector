# 集約 AI(週次トレンド/ダイジェスト)の変遷 — 思考の推移

> 散在していた複数の設計メモを、主題ごとに一本の時系列ナラティブへ束ねた歴史記録。現行仕様
> ではなく、「どう考えが推移したか」を辿るための記録。各節は当時の判断と、なぜそう動いたかを
> 過去形で残す。末尾の「当時の将来構想」は、当時まだ実現していなかった前向きな意図をそのまま
> 残したもので、その後の実現状況を断定するものではない。

## 出発点 — 個別記事の先に「今週の注目」を置く (2026-04〜)

個別記事の翻訳・分析の先に、カテゴリ単位で「今週何が起きたか」を自動生成する週次の仕組みを
構想した。汎用のディープリサーチ系ツールでは構造的にやりにくい「定点観測」を差別化軸に置き、
投資助言ではなく情報集約というポジションを取ることで、法務リスクも構造的に下げる前提だった。

## トレンド検出を別軸として切り出す

既存のキーワード機能が「この記事は何の話か(軸1=分類)」と「何が新しく・ホットになっているか
(軸2=トレンド検出)」を1つのリストで兼ねようとして、どちらも中途半端になっていた。問いも、
ユーザーの状態(関心軸が既にある/探している)も、正解の基準(分類の正確さ/検知の早さと信号
対雑音比)も別物だ、という理由でこの2つを分離した。軸1は既存分類の延長、軸2は新機能と位置
づけた。

## 転換1 — 集計を「都度計算」から「週1行の永続スナップショット」へ

当初は集計テーブルを持たず API で都度計算する方針だったが、全ユーザーに同一でリアルタイム性も
要らず、毎回集計するのは無駄だとして、週に1行のスナップショットとして永続化する形へ転換した。

## 転換2 — 件数集計から LLM の意味判断へ、ただし両者を併存させる

件数ベースの集計だけでは、一般名詞や媒体名の混入・同じ対象の表記ゆれ・古参を新顔と誤判定する、
といった限界が観測された。そこで名寄せや重要度の弁別を LLM の意味判断に委ねる方向へ動いた。
ただし「件数=事実の網羅性」と「LLM=重要度の解釈」は別の軸だとして、片方をもう片方の
フォールバックにするのではなく、常に両方を併存表示する設計にした。

## 転換3 — 生成モデルの選定が往復し、最後は単独運用へ

集約生成に使うモデルは二転三転した。当初は一つの基盤モデルを MVP に据えたが、本文を LLM に
渡さない設計が固まって規約上の懸念が無効化したため、コストの安いプロバイダを既定にし、別
プロバイダをフェイルオーバーに置く案へ移った。その後さらに、別プロバイダ切替の運用複雑性は
不要で、失敗時は件数ベースのスナップショットへ落とせば十分だとして、フェイルオーバー自体を
撤回し単独運用へ収束した。

## 転換4 — 新顔判定の根拠を「件数の振り返り」から「初出の構造」へ

「新しく出てきたもの」の判定根拠を、直近数週間の件数の振り返りから、対象を正規化(名寄せ)した
うえでの初出時刻という構造的な基準へ移した。あわせて、抽出の段で半ば分類までやらせていたのを、
抽出は「観察(見たままの記述)」に後退させ、名寄せ・正規化を別の段の責務へ移した。

## 転換5 — 移行はクリーンブレイクで

スナップショット基盤への移行は、旧テーブルを残して拡張する案とクリーンブレイク案を両論で検討
したが、移行の動機の核が旧プロンプト由来のノイズの抜本解消だったため、旧テーブルを捨てて全件
作り直す方を選んだ。

## 起きた移管(どこからどこへ動いたか)

- 「今週の注目」の作り方が、閾値チューニングによる件数集計から、LLM の意味判断と件数の併存へ移った。
- トレンド検出が、キーワード機能の一部から、独立した集約(週次トレンドのスナップショットと
  実体抽出)へ移った。その後、週次トレンドのスナップショットと実体抽出として一部が形になった。
- 集計が、API での都度計算から、週1行の永続スナップショットへ移った。
- 新顔の判定根拠が、件数の振り返りから、正規化後の初出時刻へ移った。
- 生成プロバイダが、基盤モデル単体 → 既定+フェイルオーバー → 単独運用、と動いた。

## 当時の将来構想(前向きな意図として残す)

当時、次の段階として構想されていたもの(本記録は構想として残す。実現状況は断定しない):

- 出現量の時系列(モメンタム)や、実体どうしの共起ペアからの関係性発見。
- 論調・スタンスのラベル付けによる論調シフトの追跡、関係性のネットワーク図 UI。
- 分析工程の走行頻度を上げる(週複数回での中盤の捕捉)。
- 本文を渡す用途(深掘り解説・要約・チャット)が出たときに、その箇所だけ別プロバイダを使う
  非対称構成。
- トレンド検出の実現手段(LLM 抽出/埋め込みクラスタリング/両者のハイブリッド)・品質ゲート・
  MVP の UI(新規検出一覧/急上昇/検証待ちキュー)は当時いずれも未決として残されていた。
