# Trend Detection — 最新ワード検出

> ステータス: 構想段階 (要件整理中)
> 軸: 新機能の追加

## 達成したいこと

変化の早い先端テック業界で、新しい企業名・技術名・製品名など、
まだ知らないものにいち早く気づけるようにする。

投資家が「次のトレンドに先回りする」ための情報を提供する。

## Topic Tagging との違い

| | Topic Tagging (軸1) | Trend Detection (軸2) |
|---|---|---|
| 答える問い | この記事は何の話か | 何が新しく / ホットになっているか |
| ユーザーの状態 | 関心軸が既にある | 関心軸を探している |
| 主な UI | フィルタ、記事カード | トレンド画面、急上昇リスト |
| 正解の基準 | 分類の正確さ | 検知の早さと信号対雑音比 |
| 位置付け | 既存 Keyword 機能のアップデート | 完全な新機能 |

現行の Keyword 機能はこの 2 つを 1 つのリストで兼ねようとして、
どちらも達成していない。これが「中途半端」の正体であり、分離の理由。

## 未決定事項

### 検出対象の範囲
- 企業名、技術名、製品名、人名、規格・法規制 — どこまで対象か
- 粒度の判断基準

### 実現手段
以下の選択肢が議論されたが未決定:
- **LLM 抽出**: Gemini に固有名詞を自由抽出させる
- **埋め込みベース**: 記事 embedding のクラスタリング・異常検知
- **ハイブリッド**: LLM で名前を得て、embedding で名寄せ・トレンド分析

### 品質ゲート
- 未検証のワードは集計に使わない (確定)
- ノイズ除去 (一般名詞・抽出エラー) の具体的な方法
- 承認フローの運用モデル (全件目視 vs 自動ルール)

### UI / UX
- 「新規検出一覧」「急上昇」「検証待ちキュー」のどれが MVP か
- 記事詳細に「関連ワード」として補助的に表示するか
