# C-3: フロントエンドフィルタ整理

ソースフィルタ追加 + 投資カテゴリバッジのクリッカブル化。
変更ファイル: 7（バックエンド 2 + フロントエンド 5）。DB マイグレーションなし。

## Step 1: バックエンド — sourceId フィルタ追加

■ 変更: backend/app/routers/news.py
- list_news の引数に source_id: int | None = Query(None, alias="sourceId") 追加
- WHERE 条件: NewsArticle.source_id == source_id（None でなければ）
- ※ NewsQuery スキーマは存在しない。ルーター引数に直接追加するだけ。schemas/news.py の変更は不要

■ 変更: backend/tests/test_routers/test_news.py
- sourceId フィルタのテスト追加（正常系 + 存在しない sourceId → 空配列）

## Step 2: フロントエンド型更新

■ 変更: frontend/src/types/index.ts
- NewsQuery に sourceId?: number を手動追加
- ※ NewsQuery は手動定義の型（generated.ts 由来ではない）。npm run generate-types では追加されない

## Step 3: NewsFilters に Source ドロップダウン追加

■ 変更: frontend/src/lib/api-client.ts
- getSources() SSR 関数を新規追加（既存の clientListSources() はクライアント用で SSR では使えない）

■ 変更: frontend/src/components/news/NewsFilters.tsx
- props に sources: NewsSourceResponse[] 追加
- 既存の Sentiment / Sort by / Investment / Order と並列に Source Select 追加
- デフォルト「All Sources」、選択時に ?sourceId={id} を URL パラメータに設定

■ 変更: frontend/src/app/(protected)/page.tsx
- SSR データ取得に getSources() を追加（Promise.all に含める）
- NewsFilters に sources props を渡す
- parseSearchParams に sourceId パラメータの解析追加

## Step 4: 投資カテゴリバッジのクリッカブル化

■ 変更: frontend/src/components/news/CategoryBadge.tsx
- 表示のみの <span> → クリック可能な <Link href={`/?category=${slug}`}> に変更
- クリック → URL に ?category={slug} 設定 → Investment ドロップダウン連動 → 記事一覧再フィルタ
- 他フィルタパラメータはリセットされる（カテゴリ探索の意図が明確なため問題なし）
- Server Component のまま維持（useSearchParams 不要）
