# プロンプト設計ガイドライン

> 作成日: 2026-03-20
> AI分析パイプラインで使用するプロンプトの判断基準を定義する。
> 運用しながらチューニングするドキュメントであり、ドメインモデルとは独立して改訂される。

## 1. impactLevel 判断基準

ArticleAnalysis の `impactLevel`（業界インパクト度）は enum 4段階。
LLMには `reasoning`（判定理由）を先に出力させ、その根拠に基づいて `impactLevel` を判定させる（Chain of Thought）。

判断の軸は **影響の範囲（Scope）** と **財務・ビジネスへの直結度（Financial Impact）** の2つ。

### critical（業界構造を変えるパラダイムシフト）

- **影響の範囲**: 一企業にとどまらず、業界全体、サプライチェーン全体、あるいは国家レベルに影響が及ぶ
- **ビジネス・財務指標**: 兆円規模の超大型M&A、業界トップ企業の倒産・解体、市場のルールを根底から覆す法規制
- **技術指標**: 既存技術を過去のものにするレベルのブレイクスルー（実用的な量子エラー訂正の達成、AGIの基盤技術の確立等）
- **投資家の行動**: 直ちにポートフォリオの前提を見直し、対応を検討すべきレベル
- **ニュース例**:
  - 「米国政府、先端AI半導体の輸出規制を全面強化」
  - 「TSMC台湾工場が地震で稼働停止、世界の半導体供給に致命的打撃」
  - 「Googleを反トラスト法違反で分割へ」

### high（特定企業・セクターへの明確で重大な影響）

- **影響の範囲**: 特定の企業（またはその直接の競合）の株価やシェアにダイレクトに影響を与える
- **ビジネス・財務指標**: 予想を大きく上回る/下回る決算発表、主力事業のピボット、CEOなど重要経営陣の交代、大型の資金調達や提携
- **技術指標**: ゲームチェンジャーとなり得る新製品の発表（次世代LLMのローンチ、AI創薬プラットフォームの大型臨床試験成功等）
- **投資家の行動**: 該当銘柄や関連銘柄を保有している場合、今日・今週のアクション（売買）のトリガーになり得るレベル
- **ニュース例**:
  - 「NVIDIA、次世代アーキテクチャGPUの詳細を発表」
  - 「OpenAI、新しい推論モデルをリリース」
  - 「某社、決算でガイダンスを大幅に下方修正」

### medium（業界動向として把握しておくべき進捗）

- **影響の範囲**: 企業活動の正常な範囲内での進捗や、業界のトレンドを示すもの
- **ビジネス・財務指標**: 通常の製品アップデート、中規模のパートナーシップ締結、新市場への参入発表
- **技術指標**: 既存技術の順当な性能向上、ニッチな領域での新しいユースケースの発見
- **投資家の行動**: すぐに行動を起こす必要はないが、業界の全体感や文脈を理解するためにインプットしておくべきレベル
- **ニュース例**:
  - 「Microsoft、Copilotに新機能を追加」
  - 「AWS、新しいデータセンターを日本に建設」
  - 「AIスタートアップの某社、シリーズBで資金調達」

### low（限定的な影響）

- **影響の範囲**: 企業内の局所的な出来事や、実態の伴わない情報
- **ビジネス・財務指標**: 日常的な人事異動、単なるマーケティング・PR記事、実体のない噂や観測記事
- **技術指標**: 既存製品のマイナーバグ修正、影響範囲の狭い技術解説
- **投資家の行動**: タイトルだけ読めば十分、あるいはスキップしてよいレベル
- **ニュース例**:
  - 「某社、新しいロゴデザインを発表」
  - 「アナリストが某社の目標株価を微修正」
  - 「（事実の伴わない）某社買収の噂が浮上」

## 2. LLM出力フォーマット

分析結果は以下の JSON フィールドで出力させる:

```json
{
  "title_ja": "記事タイトルの日本語翻訳（正確・簡潔）",
  "summary_ja": "3行要約。行1: 事実。行2: 業界影響。行3: 投資示唆（\\nで改行）",
  "impact_level": "low / medium / high / critical",
  "reasoning": "impactLevel の判定理由（日本語、1-2文）",
  "keywords": ["関連キーワード"]
}
```

### 出力順序（Chain of Thought）

プロンプト上では `reasoning` → `impact_level` の順で出力させる。
LLM が根拠なくスコアを出すのを防ぎ、判定精度を向上させるため。

### keywords フィールド

`keywords_by_category`（カテゴリ別のキーワード候補リスト）がプロンプトに提供された場合のみ出力。
候補リストから記事に関連するキーワードを最大3件選択させる。該当なしなら空配列。

### 入力テキスト

プロンプトに渡す記事情報:

| フィールド | 必須 | 説明 |
|-----------|------|------|
| title | はい | 記事タイトル（英語） |
| description | いいえ | RSS の要約テキスト |
| content | いいえ | 全文（trafilatura で取得、`CONTENT_MAX_LENGTH` で切り詰め） |

`content` がある場合はより深い分析が可能。ない場合は `title` + `description` でフォールバック。
