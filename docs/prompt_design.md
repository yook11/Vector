# プロンプト設計ガイドライン

> 作成日: 2026-03-20
> AI分析パイプラインで使用するプロンプトの判断基準を定義する。
> 運用しながらチューニングするドキュメントであり、ドメインモデルとは独立して改訂される。

## 1. LLM出力フォーマット

分析結果は以下の JSON フィールドで出力させる:

```json
{
  "title_ja": "記事タイトルの日本語翻訳（正確・簡潔）",
  "summary_ja": "3行要約。行1: 事実。行2: 業界影響。行3: 投資示唆（\\nで改行）",
  "reasoning": "category / topic 判定理由（日本語、1-2文）",
  "keywords": ["関連キーワード"]
}
```

### keywords フィールド

`keywords_by_category`（カテゴリ別のキーワード候補リスト）がプロンプトに提供された場合のみ出力。
候補リストから記事に関連するキーワードを最大3件選択させる。該当なしなら空配列。

### 入力テキスト

プロンプトに渡す記事情報:

| フィールド | 必須 | 説明 |
|-----------|------|------|
| title | はい | 記事タイトル（英語） |
| description | いいえ | RSS の要約テキスト |
| content | いいえ | 全文（trafilatura で取得、`CONTENT_MAX_LENGTH` で切り詰め） |

`content` がある場合はより深い分析が可能。ない場合は `title` + `description` でフォールバック。
